## Post-MVP — Gameplay & Reliability (2026-04-18)

- **Admin UX** — Category disable now shows confirmation modal. Fixed empty JSON body error on PATCH requests.
- **Disabled categories filtering** — New public endpoint `GET /api/admin/disabled-categories`. Room creation form fetches and hides disabled categories from the dropdown.
- **StubMidiProvider removed** — Server now requires Supabase (fails explicitly on startup instead of silently serving fake songs). Deleted obsolete `smoke-sprint3.ts`.
- **Empty playlist handling** — Game loop detects empty `getMidis()` result, sends `bot.noSongsAvailable` chat message, and returns to lobby instead of crashing.
- **Audio unlock moved to lobby** — Replaced blocking `StartAudioOverlay` (fullscreen z-50) with inline `AudioUnlockBanner` in the lobby. Players unlock audio before the game starts. Fallback banner in GameBoard for reconnection edge cases. Daily challenge also uses the inline banner.
- **Player ready system** — `isReady` field added to `RoomPlayer` (shared types). New `player:ready` socket event. Lobby shows ready/not-ready toggle per player, ready count badge, and host can only start when all players are ready.
