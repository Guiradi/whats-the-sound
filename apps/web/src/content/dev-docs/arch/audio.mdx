## Stack

- **`tone`** (15.1.22) — AudioContext, Transport, synths, analyser
- **`@tonejs/midi`** (2.0.28) — parser de arquivos .mid → objeto JS

## Componentes (client-side, `apps/web/src/lib/midi/`)

- `types.ts` — `MidiData`, `MidiTrack`, `MidiNote` (shape interno)
- `parser.ts` — `parseMidiFromUrl(url)` / `parseMidiFromBuffer(buf)` → `MidiData`
- `audio-context.tsx` — `AudioContextProvider` + `useAudioContext()` gate pro `Tone.start()`
- `soundfont-loader.ts` — `InstrumentRegistry`: 1 PolySynth por "slot" (melodic/drum). **Cascade fallback** (soundfont real → piano → PolySynth) termina em PolySynth até TASK-018 trazer assets reais
- `phase-player.ts` — classe `PhasePlayer` que toca uma fase: filtra tracks por `phase.tracks`, filtra notas por `[startBeat, endBeat)`, agenda em `Tone.Transport`. Stop = cancel + position=0
- `test-melody.ts` — gera um MIDI in-memory pra smoke test (C major + bass)

## Hook público

`apps/web/src/hooks/use-midi-player.ts`:

```ts
interface UseMidiPlayer {
  loadMidi(url: string): Promise<void>;
  loadMidiFromBuffer(buffer: ArrayBuffer): void;
  play(phase: PhaseConfig): Promise<void>;
  stop(): void;
  replay(): Promise<void>;
  isPlaying: boolean;
  isLoading: boolean;
  currentPhase: PhaseConfig | null;
  progress: number;      // 0-1 via rAF loop
  analyser: Analyser | null;
  midi: MidiData | null;
  error: Error | null;
}
```

**Importante**: `play()` recebe `PhaseConfig` inteiro (tracks, startBeat, endBeat), não `1|2|3|4`. Anti-cheat: client nunca tem as 4 fases em memória de uma vez. O caller (socket handler ou daily controller) entrega uma fase por vez.

## Autoplay Safari / iOS

Safari bloqueia áudio até um **user gesture**. Fluxo:

1. No **lobby** (antes do jogo), renderizar `<AudioUnlockBanner />` — banner inline não-bloqueante que pede o tap para desbloquear áudio
2. Ao clicar, chama `useAudioContext().start()` → `await Tone.start()` → badge verde "Áudio pronto!"
3. Quando o jogo inicia, o áudio já está desbloqueado para todos os jogadores

**Fallback**: Se o jogador reconectar durante o jogo sem ter desbloqueado áudio (edge case), o `GameBoard` mostra um banner flutuante no topo (z-50, não-bloqueante) para desbloquear.

**Daily Challenge**: Também usa `<AudioUnlockBanner />` inline no topo do componente.

> **Nota**: O antigo `<StartAudioOverlay />` (fullscreen bloqueante z-50) foi substituído pelo banner inline. O componente ainda existe no codebase mas não é mais importado.

## Visualizer

`apps/web/src/components/audio/audio-visualizer.tsx` — canvas-based, lê `analyser.getValue()` (Float32Array dB, normaliza de `[-85, 0]` → `[0, 1]`), 30 fps rAF throttled, pausa via Page Visibility.

Cores: gradient `--color-accent-cyan` → `--color-accent-magenta` lidas via `getComputedStyle`.
